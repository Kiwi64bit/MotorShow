Beginner Guide

Component Library Docs

This is your reference guide for all the ready-made building blocks you can use in your pages. Each component is a CSS class — you add it to an HTML element and the style is applied automatically. No JavaScript needed for most of them!

How to use any component

The golden rule: Every component works by adding a CSS class to an HTML element. A class is written like class="button primary". You can combine multiple classes on the same element (separate them with a space).

Make sure your HTML file links to the stylesheet at the top of the <head>:

<link rel="stylesheet" href="global.css">

That's it — once the stylesheet is linked, all classes in this guide are available to you.

Buttons

Buttons let users take actions — submit a form, open a menu, confirm a choice. They come in several color variants to communicate meaning.

What is it? A styled clickable element. You always start with the base class button, then add a color variant class like primary.

Base Button Required

Add class="button" to any <button> or <a> tag to get a plain unstyled-looking interactive button. 

<button class="button">Plain Button</button>

Color Variants Optional

Add a second class to change the color. Each color carries a meaning — use them consistently across your page. 

<!-- Blue — your main action -->
<button class="button primary">Primary</button>

<!-- Purple — secondary / less important action -->
<button class="button secondary">Secondary</button>

<!-- Yellow — highlight / special action -->
<button class="button accent">Accent</button>

<!-- Green — confirm / all good -->
<button class="button success">Success</button>

<!-- Orange — proceed with caution -->
<button class="button warning">Warning</button>

<!-- Red — destructive / delete -->
<button class="button danger">Danger</button>

Button on a Link

The button class also works on <a> tags — handy for "Call to Action" links that should look like buttons.

Go to page 
<a href="#" class="button primary">Go to page</a>
💡 Tip: Use primary for the most important action on a page (e.g., "Submit"). Use danger only for irreversible actions like deleting something. Using too many colored buttons on one page dilutes their meaning!
Class What it does
button Base class — always required
primary Blue background, white text
secondary Purple background, white text
accent Yellow background, dark text
success Green background, dark text
warning Orange background, dark text
danger Red background, white text

Containers

Containers center your content horizontally and prevent it from stretching too wide on large screens. Think of them as a frame that keeps everything readable.

What is it? A <div> wrapper that adds max-width and horizontal padding. You wrap your page content inside it so it doesn't run from edge to edge.

The Four Sizes

There are four size variants. Pick the one that matches how wide your content should be.

container-sm — max 540px
container-md — max 720px
container-lg — max 960px
container-xl — max 1140px
<!-- Narrow — great for article text / forms -->
<div class="container-sm">
  Your content here
</div>

<!-- Medium — blog posts, cards -->
<div class="container-md">
  Your content here
</div>

<!-- Large — dashboards, wider layouts -->
<div class="container-lg">
  Your content here
</div>

<!-- Extra large — full site layout -->
<div class="container-xl">
  Your content here
</div>
💡 Tip: On small screens (mobile), all containers behave the same — they just add a little padding on the sides. The max-width only kicks in on wider screens.
⚠️ Note: There is also a plain container class with no max-width limit. It's useful when the Navbar or Footer need to manage their inner layout, but for page content always prefer one of the sized variants above.
Class Max Width Best for
container-sm 540px Login forms, narrow articles
container-md 720px Blog posts, signup pages
container-lg 960px Dashboards, card grids
container-xl 1140px Full site wrapper

Form Controls

Four classes work together to build great-looking forms: form-group, form-label, form-control, and form-text. There are also validation state classes.

What is it? A set of classes that style form fields — text inputs, textareas, selects — along with their labels and helper messages. They work as a team: wrap each field in a form-group for proper spacing.

A Single Field — the anatomy

This is a helper message shown below the field. 
<!-- 1. Wrap everything in form-group -->
<div class="form-group">

  <!-- 2. Label -->
  <label class="form-label">Your Name</label>

  <!-- 3. The actual input -->
  <input type="text" class="form-control" placeholder="e.g. Jane Smith"/>

  <!-- 4. Optional helper text -->
  <span class="form-text">This is a helper message.</span>

</div>

Textarea

Use a <textarea> tag instead of <input> — the form-control class automatically gives it a minimum height and lets the user resize it vertically. 

<div class="form-group">
  <label class="form-label">Message</label>
  <textarea class="form-control" placeholder="Write your message..."></textarea>
</div>

Validation States

Add a state class to form-control to show the user whether their input is valid, invalid, or needs attention. Pair it with a form-feedback message for clarity.

Looks great!
Password must be at least 8 characters.
This username is already taken. 
<!-- ✅ Success state -->
<div class="form-group">
  <label class="form-label">Email</label>
  <input class="form-control is-success" type="email"/>
  <span class="form-feedback success">Looks great!</span>
</div>

<!-- ❌ Error state -->
<div class="form-group">
  <label class="form-label">Password</label>
  <input class="form-control is-error" type="password"/>
  <span class="form-feedback error">Too short!</span>
</div>

<!-- ⚠️ Warning state -->
<div class="form-group">
  <label class="form-label">Username</label>
  <input class="form-control is-warning" type="text"/>
  <span class="form-feedback warning">Already taken.</span>
</div>
Class Put it on What it does
form-group <div> Wrapper — adds spacing between fields
form-label <label> Styles the field label
form-control <input>, <textarea>, <select> Styles the input itself
form-text <span> or <p> Small grey helper text below the field
is-success form-control element Green border (valid)
is-error form-control element Red border (invalid)
is-warning form-control element Orange border (needs attention)
form-feedback success/error/warning <span> or <p> Colored feedback text matching the state

Navbar

The navbar sits at the top of your page. It holds your brand name, navigation links, and optionally a theme toggle.

What is it? A horizontal bar that runs across the top of the page. It's made from a <nav> tag with class navbar containing a brand name, a nav list, and optional extras.

Full Navbar

MyBrand
  • Home
  • About
  • Contact
  • Sign Up
<nav class="navbar">
  <div class="container">

    <!-- Brand / logo -->
    <a href="/" class="navbar-brand">MyBrand</a>

    <!-- Nav links -->
    <div class="navbar-nav">
      <ul class="nav">
        <li><a href="#" class="nav-link">Home</a></li>
        <li><a href="#" class="nav-link">About</a></li>
        <li><a href="#" class="nav-link">Contact</a></li>

        <!-- You can mix in a button -->
        <li><a href="#" class="button primary">Sign Up</a></li>
      </ul>
    </div>

  </div>
</nav>

Theme Toggler Optional

The theme-toggler is a pill-shaped toggle button you can drop anywhere — most commonly inside the navbar. It needs a small JavaScript snippet to actually switch themes (see below).

← Click it! 
<!-- The button -->
<button class="theme-toggler" id="themeToggler"></button>

<!-- The JavaScript to make it work -->
<script>
  // Grab the button and the html element
  const toggler = document.getElementById('themeToggler');
  const html = document.documentElement;

  toggler.addEventListener('click', () => {
    // Switch between "light" and "dark"
    const current = html.getAttribute('data-theme');
    html.setAttribute('data-theme', current === 'dark' ? 'light' : 'dark');
  });
</script>
Class Put it on What it does
navbar <nav> The navbar container bar
navbar-brand <a> Your brand / logo text (big, styled font)
navbar-nav <div> Wrapper that grows to fill available space
nav <ul> The list of navigation links
nav-link <a> inside <li> A single navigation link
theme-toggler <button> Light/dark mode pill toggle

Footer

The footer sits at the bottom of your page. It has a brand blurb on the left, a group of links in the middle/right, and a copyright line at the bottom.

What is it? A structured bottom section built with footer-brand, footer-links, and footer-bottom inside a <footer> element.

Full Footer

MyBrand

A short description of what your website or product is about. Keep it to 1–2 sentences.

Home About Privacy Contact

Pattern: Full Page Skeleton

This is how all components come together to build a complete HTML page — navbar at the top, content in a container, footer at the bottom. 

<!DOCTYPE html>
<html lang="en" data-theme="light">
<head>
  <meta charset="UTF-8"/>
  <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
  <title>My Page</title>
  <link rel="stylesheet" href="global.css">
</head>
<body>

  <!-- ① Navbar -->
  <nav class="navbar">
    <div class="container">
      <a href="/" class="navbar-brand">MyBrand</a>
      <div class="navbar-nav">
        <ul class="nav">
          <li><a href="#" class="nav-link">Home</a></li>
          <li><a href="#" class="nav-link">About</a></li>
          <li><button class="theme-toggler" id="themeToggler"></button></li>
        </ul>
      </div>
    </div>
  </nav>

  <!-- ② Main Content -->
  <main>
    <div class="container-lg">
      <h1>Welcome to My Page</h1>
      <p>Your content goes here.</p>
      <a href="#contact" class="button primary">Get in Touch</a>
    </div>
  </main>

  <!-- ③ Footer -->
  <footer class="footer">
    <div class="container">
      <div class="footer-brand">
        <h3>MyBrand</h3>
        <p>Short description.</p>
      </div>
      <div class="footer-links">
        <a href="#">Home</a>
        <a href="#">Privacy</a>
      </div>
    </div>
    <div class="footer-bottom">
      <p>© 2025 MyBrand.</p>
    </div>
  </footer>

  <!-- ④ Theme toggle script -->
  <script>
    const toggler = document.getElementById('themeToggler');
    toggler.addEventListener('click', () => {
      const html = document.documentElement;
      const current = html.getAttribute('data-theme');
      html.setAttribute('data-theme', current === 'dark' ? 'light' : 'dark');
    });
  </script>

</body>
</html>
💡 Notice: The data-theme="light" attribute on <html> is what activates the light or dark theme. The JavaScript in the theme toggler swaps it between "light" and "dark".

CSS Variables

The stylesheet defines a set of design tokens — named values for colors, spacing, font sizes, and fonts — that you can use in your own CSS. They ensure your custom styles always match the rest of the design.

What is a CSS variable? It's a name that holds a value. Instead of writing color: hsl(230, 87%, 60%) over and over, you write color: var(--primary-color). If the design ever changes, it updates everywhere automatically. Variables always start with two dashes: --.
How to use them: In your own .css file, reference any variable with var(--variable-name). They are available everywhere because they are defined on :root — meaning the whole page. 
/* In your own stylesheet — use any variable like this: */
.my-card {
  background-color: var(--surface-1);
  border: 1px solid var(--border-color);
  color: var(--text-color);
  padding: var(--spacing6);
  font-family: var(--ff1);
  font-size: var(--fs5);
}

Brand Colors

These are the core colors of the design system. Use them to keep your custom elements visually consistent with the components.

--primary-color Blue — main actions, links, focus rings
--secondary-color Purple — secondary actions
--tertiary-color Teal — accents, highlights
--accent-color Yellow — special callouts

Background & Surface Colors

These three variables form a layering system. Use them to create depth — like a card sitting on top of a page background.

--base-color The page background itself
--surface-1 Cards, sidebars, inputs
--surface-2 Hover states, table stripes

Text Colors

Always use these variables for text — never hardcode a color like #000. This is what makes dark mode work automatically.

--text-color Main body text
--text-muted Captions, helper text, labels
--text-dark Always dark (e.g. text on yellow buttons)
--text-light Always light (e.g. text on blue buttons)

Status Colors

Use these to communicate meaning in your UI — same colors used by buttons and form validation states.

--success-color Green — valid, complete
--warning-color Orange — caution needed
--danger-color Red — error, destructive
--info-color Cyan — informational

Border Color

--border-color Lines, dividers, input borders

Font Sizes

Six responsive sizes. --fs1 through --fs4 grow slightly on larger screens (they use vw units under the hood). --fs5 and --fs6 are fixed. In practice: use --fs6 for body text and --fs1–--fs4 for headings.

--fs1 Heading 1 ~34px, responsive
--fs2 Heading 2 ~30px, responsive
--fs3 Heading 3 ~28px, responsive
--fs4 Heading 4 ~24px, responsive
--fs5 Subheading 1.25rem (20px), fixed
--fs6 Body / UI text 1rem (16px), fixed 
/* Examples */
.page-title    { font-size: var(--fs1); }  /* big hero heading */
.section-title { font-size: var(--fs3); }  /* section subheading */
.caption       { font-size: var(--fs6); }  /* normal body / UI text */

Font Families

Two font stacks are available. The stylesheet already applies them to the right elements — but you can use them in your own CSS too.

var(--ff1)

Putting It All Together — Custom Card

Here is a custom card component built entirely using design tokens. It will automatically adapt to light and dark mode because it uses variables instead of hardcoded colors.

Card Title

This card uses only design tokens. Toggle dark mode (top right) to see it adapt instantly. 

/* In your CSS file */
.card {
  background: var(--surface-1);
  border: 1px solid var(--border-color);
  border-radius: .75rem;
  padding: var(--spacing6);
  font-family: var(--ff2);
}

.card-title {
  font-family: var(--ff1);
  font-size: var(--fs5);
  font-weight: 600;
  color: var(--text-color);
  margin-bottom: var(--spacing2);
}

.card-body {
  font-size: var(--fs6);
  color: var(--text-muted);
  margin-bottom: var(--spacing5);
}
💡 Dark mode for free: Because the stylesheet redefines variables like --surface-1 and --text-color when data-theme="dark" is set, any element that uses these variables will automatically update when the user switches themes — with zero extra work from you.
